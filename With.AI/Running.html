실행 하기

실행 하기

라즈베리파이에 적용하기

생성된 hef 파일로 gstreamer를 이용하여 카메라 입력에 바로 AI 모델 적용이 가능합니다. tappas 예제를 이용하여 쉽게 제작할 수 있습니다. 개발 환경 리눅스에 첨부파일에 있는 tappas_3.23.0_linux_installer.zip 파일의 압축을 풀어줍니다.

Tappas 이용하기

tappas[version] 폴더의 내부에는 apps라는 폴더가 있는데 해당 폴더가 Tappas 예제가 있는 폴더입니다. 내부에는 7개 정도의 예제 파일이 있으며 해당 문서에서는 Detection예제를 설명합니다.   각 예제파일의 내부에는 gstreamer명령어가 작성되는 쉘 스크립트가 존재합니다.

./<예제>.sh -h 를 실행하면 해당 쉘 스크립트의 인자 값들이 어떻게 구성되어 있는지 알 수 있고 --print-gst-launch 명령어를 통해 해당 예제의 파이프라인이 어떻게 구성되는지 알 수 있습니다.

Detection의 파이프라인은 gst-launch-1.0 v4l2src device=/dev/video0 ! – 카메라 지정 video/x-raw,format=YUY2,width=1280,height=720,framerate=30/1 ! – Caps 지정 (gst-device-monitor-1.0 명령어로 확인 가능) queue leaky=downstream max-size-buffers=5 max-size-bytes=0 max-size-time=0 ! hailonet hef-path=/home/root/apps/detection/resources/yolov5m_yuv.hef is-active=true !

  • hailonet을 이용하여 hef 모델 파일을 파이프라인에 적용 queue leaky=no max-size-buffers=30 max-size-bytes=0 max-size-time=0 ! hailofilter function-name=yolov5 config-path=/home/root/apps/detection/resources/configs/yolov5.json so-path=/usr/lib/hailo-post-processes/libyolo_post.so qos=false !

  • hailofilter는 모델 config내용이 있는 json파일과 결과값을 처리할 post_processes 파일을 지정하여 파이프라인에 적용 queue leaky=no max-size-buffers=30 max-size-bytes=0 max-size-time=0 ! hailooverlay ! queue leaky=downstream max-size-buffers=5 max-size-bytes=0 max-size-time=0 ! videoconvert ! fpsdisplaysink video-sink=autovideosink name=hailo_display sync=false text-overlay=false – autovideosink를 이용하여 연결된 LCD로 출력 가능합니다. Rtspclientsink를 이용하면 Rtsp streaming이 가능합니다. 이렇게 구성되어 있습니다.   Hailo Element에 대한 자세한 내용은 “gst-inspect-1.0 [ElementName]” 명령어로 확인 가능합니다. Ex) gst-inspect-1.0 | grep hailo, gst-inspect-1.0 hailonet, gst-inspect-1.0 hailofilter etc.

Post-process 작성

gstreamer는 Meson 빌드 시스템을 이용합니다. Hailo tappas_v3.22.0\core\hailo\libs\postprocesses 경로에서 Post Processes에 대한 소스 및 Meson.build 파일을 확인할 수 있습니다.

Meson.build 파일을 열어보면 각 폴더별로 so파일을 빌드하는 내용들이 적혀 있으며 적용할 모델에 맞는 Post Process를 골라서 수정하면 됩니다.

detection 예제의 경우 yolo_postprocess.cpp 소스를 사용합니다. 커스텀 소스(첨부된 my_post.cpp / my_post.hpp)는 아래처럼 Meson.build 에 추가하여 빌드하여 사용하면 됩니다.

소스 빌드는 tappas 경로(압축해제하여 사용하는 tappas workspace)에서 쉘 스크립트를 이용해서 빌드 합니다.

./scripts/gstreamer/install_hailo_gstreamer.sh

※ meson 에러가 발생하는 경우

  1. Root로 변환(sudo su) 하여 pip3 install meson으로 meson을 재설치합니다.
  2. PATH=$PATH:/home//.local/bin 명령어로 meson 경로를 환경변수에 등록합니다.
  3. Export TAPPAS_WORKSPACE= 로 지정합니다.
  4. 다시 ./scripts/gstreamer/install_hailo_gstreamer.sh를 실행하면 됩니다. My_post를 빌드한 경우 간단한 테스트는 아래 커맨드로 테스트 가능합니다.
    gst-launch-1.0 videotestsrc ! hailofilter so-path=/home/pi/hailo_install/tappas_v3.24.0/apps/h8/gstreamer/libs/post_processes/libmy_post.so ! autovideosink

Json 파일 작성

Tappas의 예제 폴더에는 각 모델에 맞는 configs 폴더가 존재합니다. Ex) detection 폴더 내 config 파일들

Hailofilter 에서 사용하는 json 파일에는 iou, threshold, max_boxes, anchors, labels등의 값이 있습니다.

  • Yolov5.json 내용 아래 내용을 참조하여 파일을 수정하면 됩니다.
    • iou_threshold : iou 값
    • detection_threshold : detection 되었음을 판단할 최소 값
    • output_activation : 활성화 함수
    • label_offset : 레이블 시작위치 지정 값
    • max_boxes : 동시 생성할 박스의 최대 개수
    • anchors : yolo 에서 사용하는 anchors 값
    • labels : 라벨링 작업 때 지정된 라벨 이름 리스트이며 맨 앞에 unlabeled 는 고정입니다.
      "iou_threshold": 0.45,
"detection_threshold": 0.3,
"output_activation": "none",
"label_offset": 1,
"max_boxes": 200,
“anchors": [ [116, 90, 156, 198, 373, 326], [30, 61, 62, 45, 59, 119], [10, 13, 16, 30, 33, 23] ],
"labels": [ "unlabeled", "person", "bicycle", "car", "motorcycle", "airplane", "bus", "train", "truck" ]
