DFC 튜토리얼

DFC 튜토리얼

hailo 칩에서 일반 PC환경에서 생성한 모델을 인식할 수 있도록 하기 위해서 HEF(hailo Executable File)파일로 변환해야 합니다.

  • 1번 parsing
  • 2번 optimization
  • 4번 compilation

튜토리얼 파일을 참고하면 됩니다.

추가로 첨부된 yolov5s_convert_hef_DEMO.ipynb 파일은 튜토리얼 내용을 하나로 합친 파일입니다. 아래 부터는 yolov5s_convert_hef_DEMO.ipynb 파일 기준으로 설명합니다.

모델 Parsing

파싱 튜토리얼에서는 사용할 모델을 HAR(Hailo Archive) 파일로 변경하는 방법을 설명합니다. 지원하는 모델은 ckpt, onnx, pb, Tensorflow Lite 파일 형식입니다. 결과물(har파일)을 저장하면서 진행합니다.

라이브러리 Import

  • 아래 주석된 2줄은 필요하면 실행합니다.
  • 3번 박스를 실행할 때 onnxsim이 필요하다고 하는 경우가 있습니다. onnx 파일의 경로와 모델 정보를 입력하여 har 파일로 변환합니다.

※ Start Node와 End Node는 AI모델을 Hailo가 인식할 수 있도록 변환하는 시작점과 끝점의 기준입니다. Onnx로 되어있는 AI 모델을 Netron이라는 프로그램을 설치하여 열어보면 노드를 시각화 하여 보여줍니다. 제일 위에 있는 노드를 StartNode로 지정하고 변환 하고싶은 마지막 노드를 EndNode로 지정하면 됩니다. 파싱된 har 파일을 저장해주시면 다른 튜토리얼 노트북 파일에서 사용 가능합니다.

모델 Optimization

  • 모델 최적화 튜토리얼 에서는 파싱된 har 파일을 기반으로, calibration 데이터셋과, 모델 스크립트인 alls파일로 최적화를 진행합니다.
  • 필요한 패키지를 임포트하고 이전 튜토리얼에서 사용한 model_name으로 생성한 har 파일을 불러옵니다.
  • 아래는 calibration dataset을 생성하는 과정입니다. preproc이라는 이미지 전처리 함수를 만들고 학습할 때 사용한 이미지들을 이 preproc함수를 이용해서 데이터셋을 생성합니다.
  • alls파일은 최적화 및 컴파일 프로세스를 제어하는 명령들이 포함된 모델 스크립트를 지칭합니다. https://github.com/hailo-ai/hailo_model_zoo/tree/master/hailo_model_zoo/cfg/alls 해당 경로에서 모델별로 참고하시면 됩니다. 추가적인 설명은 DFC_2_Model_Optimization_Tutorial 내에 Model modifications 부분을 확인해주시면 됩니다.
  • Optimize 결과 Report를 저장할 csv_path를 지정하고 모델을 저장할 har 파일경로를 지정합니다. 튜토리얼에서는 alls_lines를 작성하고 저장해서 load_model_script를 실행합니다. alls파일을 다운로드 받으면 바로 load_model_script를 사용하면 됩니다.
    • Optimize를 진행하면 runner 모델에 alls_lines에 정의한 normalization1이라는 레이어가 추가되는데 Optimize중 에러가 발생하면 생성된 채로 멈추게 되어 runner 모델을 다시 불러와야 합니다.
    • runner = ClientRunner(hw_arch='hailo8', har_path=hailo_model_har_name) - 코드 재실행 최적화가 완료된 모델은 아래 코드로 검증 가능합니다.

모델 Compilation

컴파일 튜토리얼은 최적화 완료된 har 파일을 hef 파일로 저장하는 튜토리얼입니다.

  • 패키지를 선언하고 모델을 불러옵니다.
  • 불러온 모델로 컴파일하고 저장합니다.
  • 이 후 커스텀 모델을 hailo에서 사용할 수 있게 hef 파일로 변환이 완료된 것을 확인할 수 있습니다.
