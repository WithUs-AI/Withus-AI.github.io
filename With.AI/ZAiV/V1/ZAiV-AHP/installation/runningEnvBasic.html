1. 실행 환경

1.1. Withus 제공 이미지 사용 설치

  • Raspberry PI CM4

    • RaspBerry PI Imager와 사전 설치 이미지(기본 환경 구성본)를 준비합니다.

    • Raspberry PI Imager 설치프로그램의 안내에 따라 설치합니다.

    • Micro SD 연결. MicroSD카드를 Reader기에 결합 후 PC에 연결합니다.

    • 이미지를 Flashing

      • Raspberry PI Imager 실행
      • 운영체제 버튼 클릭 > 사용자 정의 사용 > 파일 선택 다이얼로그에서 다운로드받은 이미지 선택
      • 저장소 버튼 클릭 > PC에 연결한 Micro SD카드 volume을 선택.
      • 쓰기
  • Raxda CM5 (To be supported)

1.2. Clean OS 및 프로그램 설치

1.2.1. OS 설치

  • X86/AMD64

    Ubuntu 20.04 버전을 설치합니다. 아래 설치가이드를 참고해주세요.

      https://ubuntu.com/tutorials/install-ubuntu-desktop#1-overview

  • Raspberry PI CM4

    • 이미지 준비

      • RaspBerry PI Imager(x86, MacOS, Ubuntu)와 RaspbianOS Buster를 준비합니다.
        • Raspberry PI Imager Manual Download:
          • http://downloads.raspberrypi.org/imager/ 접속
          • 설치환경의 최신버전 실행파일 다운로드
        • Buster Image Manual Download:
          • http://downloads.raspberrypi.org/raspios_arm64/images/raspios_arm64-2021-05-28/ 접속
          • 2021-05-07-raspios-buster-arm64.zip 다운로드

    • Raspberry PI Imager 설치프로그램의 안내에 따라 설치합니다.

    • Micro SD 연결. MicroSD카드를 Reader기에 결합 후 PC에 연결합니다.

    • 이미지 Flashing

      • Raspberry PI Imager 실행
      • 운영체제 버튼 클릭 > 사용자 정의 사용 > 파일 선택 다이얼로그에서 다운로드받은 이미지 선택
      • 저장소 버튼 클릭 > PC에 연결한 Micro SD카드 volume을 선택.
      • 쓰기
  • Raxda CM5 (To be supported)

1.2.2. 기본 연결 및 부팅

  • 지원가능 장치 리스트를 확인하여 ZAiV-AHP에 AP Board를 연결합니다.
  • 이미지가 설치된 Micro SD카드를 Micro SD카드 슬롯에 삽입합니다.
  • 모니터의 전원을 연결하고 모니터와 장치 간에 HDMI to micro HDMI 케이블을 연결합니다.
  • USB 키보드/마우스를 연결합니다.
  • 인터넷이 가능한 Etherent Cable을 RJ45 포트에 연결합니다.
  • USB Webcam을 연결합니다.
  • 마지막으로 USB Type-C Power Adapter를 연결하여 부팅합니다.

  • gcc 9 설치 (라즈베리 파이의 경우만 따로 설치)

    • Download Binary

      준비 섹션의 링크를 통해서 다운로드받거나, 아래 링크를 통해 다운로드 받는다.

        https://sourceforge.net/projects/raspberry-pi-cross-compilers/files/Bonus%20Raspberry%20Pi%20GCC%2064-Bit%20Toolchains/Raspberry%20Pi%20GCC%2064-Bit%20Native-Compiler%20Toolchains/Buster/GCC%209.4.0/native-gcc-9.4.0-pi_64.tar.gz/download

    • 압축해제

        tar xf native-gcc-9.4.0-pi_64.tar.gz

    • 설정

        sudo mv native-pi-gcc-9.4.0-64 /opt

    • Path 설정

        echo 'export PATH=/opt/native-pi-gcc-9.4.0-64/bin:$PATH' >> .profile
  echo 'export LD_LIBRARY_PATH=/opt/native-pi-gcc-9.4.0-64/lib:$LD_LIBRARY_PATH' >> .profile
  source .profile

    • Important Symlinks 설정

        sudo ln -sf /usr/include/aarch64-linux-gnu/asm/* /usr/include/asm
  sudo ln -sf /usr/include/aarch64-linux-gnu/gnu/* /usr/include/gnu
  sudo ln -sf /usr/include/aarch64-linux-gnu/bits/* /usr/include/bits
  sudo ln -sf /usr/include/aarch64-linux-gnu/sys/* /usr/include/sys
  sudo ln -sf /usr/include/aarch64-linux-gnu/openssl/* /usr/include/openssl
  sudo ln -sf /usr/lib/aarch64-linux-gnu/crtn.o /usr/lib/crtn.o
  sudo ln -sf /usr/lib/aarch64-linux-gnu/crt1.o /usr/lib/crt1.o
  sudo ln -sf /usr/lib/aarch64-linux-gnu/crti.o /usr/lib/crti.o
  sudo ln -s /opt/native-pi-gcc-9.4.0-64/bin/aarch64-linux-gnu-g++-9.4.0   /opt/native-pi-gcc-9.4.0-64/bin/g++-9
  sudo ln -s /opt/native-pi-gcc-9.4.0-64/bin/aarch64-linux-gnu-gcc-9.4.0   /opt/native-pi-gcc-9.4.0-64/bin/gcc-9

    -DINSTALL_PYTHON_EXAMPLES=ON \

num_cores_to_use=$(($(nproc)/2))

마무리

기본 설치는 완료되었습니다. 이 후 재부팅 합니다.

  • 참고

    • PCI의 경우 재 부팅 후 hailo_pci Driver를 올려야합니다.

      sudo modprobe hailo_pci

    • 부팅 시 자동 실행하도록 하는 경우, rc파일에 등록합니다.

      • Bash Shell의 경우

          echo "modprobe hailo_pci" >> ~/.bashrc

      • zsh의 경우

          echo "modprobe hailo_pci" >> ~/.zshrc
